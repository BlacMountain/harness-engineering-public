# Exec Plans

用途：记录复杂任务“这次怎么做”，包括步骤、风险、验证方式和执行结果。

什么时候写：

- 任务跨多个文件、模块、服务或文档。
- 实施前需要固定验收标准、风险和顺序。
- 任务可能中断，需要后续 Agent 能接手。

必填内容：

- 背景。
- 目标和非目标。
- 验收标准。
- 实施步骤。
- 风险和回滚方式。
- 验证命令。
- 决策日志。

使用方式：

- 进行中的计划放入 `docs/exec-plans/active/`。
- 完成后移动到 `docs/exec-plans/completed/`，并补充结果和验证记录。
- 剩余问题进入 `docs/exec-plans/tech-debt-tracker.md`。

不要写：

- 长期产品规格。
- 永久架构说明。
- 外部资料原文复制。
