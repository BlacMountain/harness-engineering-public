# Decisions

用途：记录会长期影响项目的选择，以及为什么这样选择。

什么时候写：

- 架构、数据流、依赖、部署或安全边界发生重要取舍。
- 有多个可行方案，需要保留被拒绝方案和原因。
- 决策会影响后续 Agent 的默认实现方向。

必填内容：

- Status：proposed、accepted、superseded 或 rejected。
- Context：背景和约束。
- Decision：最终选择。
- Alternatives：考虑过的替代方案。
- Consequences：影响、风险和后续动作。

不要写：

- 一次性任务步骤。
- 普通 bug 修复记录。
- 没有长期影响的临时选择。
