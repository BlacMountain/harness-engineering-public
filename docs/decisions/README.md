# Decisions

`docs/decisions/` 记录会长期影响项目方向的架构、策略或治理决策。

## 写入条件

- 架构、数据流、依赖、部署、安全边界或文档结构发生重要取舍。
- 存在多个可行方案，需要保留被拒绝方案及原因。
- 决策会影响后续 Agent 的默认实现路径。

## 必填内容

- Status：`proposed`、`accepted`、`superseded` 或 `rejected`。
- Context：背景、约束和问题。
- Decision：最终选择。
- Alternatives：考虑过的替代方案。
- Consequences：影响、风险和后续动作。

## 禁止内容

- 不记录一次性任务步骤。
- 不记录普通 bug 修复流水账。
- 不记录没有长期影响的临时选择。
