# References

`docs/references/` 归属开发依据管理，记录外部来源、本地摘要、协议说明、第三方 API 文档链接和平台约束。

## 写入条件

- 实现或 policy 依赖外部系统、标准、论文、API 或平台文档。
- 需要保留来源、确认日期、版本或适用范围。
- 外部资料会影响长期设计、配置、验证或安全判断。

## 必填内容

- Source：来源链接、文件路径或系统名称。
- Date：记录或确认日期。
- Summary：与本项目相关的简短摘要。
- Applies to：适用范围。
- Notes：限制、风险、版本差异或后续验证事项。

## 文档边界

开发过程中依赖的外部 API、平台、协议、论文或标准资料归属 `docs/references/`。目标项目自己产出的 API 文档、用户手册、部署说明、SDK 文档、operator docs 和产品说明归属目标项目团队约定的交付文档位置。

## 禁止内容

- 不复制大段受版权保护的原文。
- 不记录没有来源的推测。
- 不保存 secret、token、cookie、private key 或私有凭据。
