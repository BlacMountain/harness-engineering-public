# References

用途：记录外部依据，包括 API、协议、论文、平台文档、工具说明和本地摘要。

什么时候写：

- 实现依赖外部系统、标准、论文或 API。
- 需要保留来源、访问日期或适用范围。
- 外部资料会影响长期设计、配置或验证方式。

必填内容：

- Source：来源链接、文件路径或系统名称。
- Date：记录或确认日期。
- Summary：与本项目相关的简短摘要。
- Applies to：适用范围。
- Notes：限制、风险或版本差异。

不要写：

- 大段复制受版权保护的原文。
- 没有来源的推测。
- secrets、token、cookie 或私有凭据。
