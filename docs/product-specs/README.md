# Product Specs

`docs/product-specs/` 归属开发规格管理，记录系统目标、行为、约束和验收标准。

## 写入条件

- 新项目或新能力需要明确目标、非目标和验收标准。
- 变更会影响用户流程、外部行为、业务规则或可见输出。
- 需求存在歧义，需要先固定行为边界再实施。

## 必填内容

- 背景和目标。
- 非目标。
- 使用者、角色或主要场景。
- 功能需求和约束。
- 可验证的验收标准。
- 开放问题和待确认事项。

## 文档边界

项目交付文档服务目标项目的用户、部署者或运维者。API 文档、用户手册、部署说明、SDK 文档、operator docs 和产品说明归属目标项目团队约定的交付文档位置。产品规格负责固定开发输入和验收边界。

## 禁止内容

- 不写本次任务的逐步实施计划。
- 不写临时调试记录。
- 不写与产品行为无关的内部实现细节。

## 验证要求

- 验收标准必须能通过测试、命令、截图、日志或人工检查验证。
