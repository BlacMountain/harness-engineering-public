# Product Specs

用途：说明系统应该是什么，包括目标用户、核心流程、需求、非目标和验收标准。

什么时候写：

- 新项目需要明确产品或能力边界时。
- 新功能会改变用户流程、业务规则或外部行为时。
- 需求不清晰，需要先固定目标和非目标时。

必填内容：

- 背景和目标。
- 非目标。
- 用户、角色或使用场景。
- 功能需求和约束。
- 可验证的验收标准。
- 开放问题。

不要写：

- 本次任务的逐步实施计划。
- 临时调试记录。
- 架构内部实现细节，除非它影响产品行为。

完成后验证：

- 验收标准能被测试、命令、截图、日志或人工检查验证。
- 与 `ARCHITECTURE.md` 和 `.harness/*.yaml` 不冲突。
